The stable — tier winners

Downstream agents request a tier, not a model. Each tier owns a benchmark and a quality threshold; the procurement agent keeps the cheapest adequate model in the saddle.

For apps — the recommendation endpoint

Any app or agent can resolve a tier to the currently recommended model. CORS-open, read-only, cached 60s server-side; cache ~1h on your side and walk fallbacks in order when the winner errors.

curl -s https://dashyng.com/api/public/agentstable/tiers
{ "tiers": { "workhorse": { "model": "…", "fallbacks": ["…"], "priceIn": 1, "priceOut": 5 }, … } }
