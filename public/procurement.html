Agent Procurement Agent (APA)

APA is the stable's procurement brain. It scrapes model-release, pricing, and benchmark news on its own dedicated feed (cheapest qualified Claude model — never grok, which is reserved for X), and acts:
  • Top-5 US labs (Anthropic, OpenAI, Google, xAI, Meta) — a new release is auto-tested against the current default on a fixed benchmark, cross-provider (each model runs on its own lab's API; a model that can't be run is never adopted blind). If it's equal-or-better and cheaper, APA auto-adopts it for that module and flags the owner with the projected $/month saving. Toggles + the benchmark board live on the agents dashboard.
  • Hosting (Together.AI, Fireworks, Groq, …) and non-US labs (DeepSeek, Qwen, Mistral, …) — surfaced as arbitrage proposals for the owner's review, not auto-adopted.
Architectural boundary: APA writes every finding + decision to the APA Feed tab; the news feed is a read-only consumer (the "Model Watch" lane) — so APA works standalone and the news feed never double-searches for model news. The nightly CI back-propagates realized savings into a per-source scoreboard, upweighting whatever surfaced an adopted change. Auto-adopt is reversible per module (dashboard → APA → revert).

Selection policy (implemented)

1. Route by task complexity. Simple/mechanical (summarize, extract, dedup, format) → Haiku. Multi-step reasoning, planning, judgment over the day → Sonnet. Both default to Claude on subscription (pre-2026-06-15) or the $100 agent credit (after).
2. Escalate on evidence, not by default. An agent escalates Sonnet→Opus only when it flags genuine ambiguity or stakes (e.g. a high-stakes reply draft). Every escalation writes a Decisions-tab row: actor, decision, why, cost. Opus runs on the API key (escalation is rare; not worth a quota request).
3. Hard-stop resilience. If the Claude agent-credit pool is exhausted (post-June-15 behavior), text generation falls over to Vertex Gemini automatically and logs the fallback. The dashboard health panel surfaces it.
4. CI tunes the routing. The nightly CI agent reads Decisions + Usage and proposes re-routing (e.g. "this class of task ran fine on Haiku 12× — demote from Sonnet"). The owner validates material changes.
5. Cost coloring. Red = charged (out-of-pocket: claw / Opus-esc on the API key). Green = credit drawdown (GCP credits for Vertex/VM; the $100 Claude agent pool after June 15). Grey = subscription-included (Claude headless before June 15) and totals. Mixed amounts show grey total + colored parts.

Open levers (not yet automated)

  • Prompt caching on the heartbeat — its ~1M-token input is the biggest single cost; caching the static spec/journal context could cut it 5–10×.
  • An API-purchasing agent that proposes provider/model swaps for the owner to validate (roadmap).
  • Gemini-vs-Haiku/Sonnet head-to-head eval, using the claw as the bed.
